## 2. 测试范围

### 2.1 MVP核心功能（15个特性）

#### 任务管理（5个）
1. ✅ 创建任务（标题、描述、优先级、截止时间）
2. ✅ 分配任务给单个Agent
3. ✅ 任务状态流转（待办/进行中/已完成/已验收/已驳回）
4. ✅ 更新任务进度（百分比）
5. ✅ 查询任务列表（按状态筛选）

#### 进度跟踪（3个）
6. ✅ 任务看板（四列布局：待办/进行中/已完成/已验收）
7. ✅ 任务卡片显示（标题、负责人、优先级、截止时间、进度）
8. ✅ 延期任务高亮显示

#### 通知提醒（3个）
9. ✅ 新任务分配通知（Agent接收）
10. ✅ 任务完成通知（管理者接收）
11. ✅ 任务驳回通知（Agent接收）

#### Agent管理（2个）
12. ✅ Agent列表（名称、类型、能力标签、状态）
13. ✅ Agent负载显示（当前任务数）

#### 协作功能（2个）
14. ✅ 任务评论（文本评论，不支持@提及）
15. ✅ 附件上传下载（基础支持）

### 2.2 非功能测试

#### 性能测试
- 页面首屏加载时间 < 3秒
- API响应时间 < 500ms
- 50个Agent轮询时系统响应正常
- 100个并发任务时性能无明显下降
- **Redis缓存命中率 > 60%**（PRD v2.2新增）
- **ETag缓存命中率 > 60%**（PRD v2.2新增）
- **304响应正确性验证**（PRD v2.2新增）
- **限流策略验证**（管理者60次/分钟、Agent10次/分钟）（PRD v2.2新增）

#### 定时任务测试（PRD v2.2新增）
- 预警扫描任务（每小时）：24h/12h/2h预警时间点准确性
- Agent心跳检测（每5分钟）：15分钟超时标记离线
- 通知清理任务（每天凌晨2点）：删除90天前已读通知
- 预警去重策略验证：Redis键过期机制

#### 安全测试
- JWT Token认证
- Agent API Token认证
- 权限控制验证
- 敏感操作二次确认

#### 兼容性测试
- 主流浏览器：Chrome、Firefox、Safari、Edge
- 响应式设计验证

```

│   ├── 负载测试
│   ├── 压力测试
│   └── 并发测试
├── 安全测试（10%）

    ├── 浏览器兼容性
    └── 响应式设计
```

### 3.2 测试方法

#### 手工测试（60%）
- **优点**：灵活、发现隐藏bug、用户体验验证
- **适用**：功能测试、UI测试、探索性测试
- **工具**：TestRail、Jira

#### 自动化测试（40%）
- **优点**：快速回归、可重复、覆盖率高
- **适用**：API测试、性能测试、回归测试
- **工具**：
  - API测试：Postman、Jest
  - E2E测试：Playwright、Cypress
  - 性能测试：JMeter、k6

## 7. Bug管理

### 7.1 Bug等级定义

| 等级 | 定义 | 响应时间 | 修复时间 |
|------|------|----------|----------|
| P0 - 阻塞性 | 系统崩溃、数据丢失、安全漏洞 | 1小时 | 4小时 |
| P1 - 严重 | 核心功能无法使用 | 4小时 | 24小时 |
| P2 - 一般 | 功能缺陷但有替代方案 | 24小时 | 48小时 |
| P3 - 轻微 | UI问题、文案错误 | 48小时 | 72小时 |

### 7.2 Bug报告模板

```markdown
## Bug标题

**严重程度**：P0/P1/P2/P3
**优先级**：高/中/低
**发现时间**：YYYY-MM-DD HH:mm
**发现人**：claw2-qa

**环境信息**：
- 测试环境：DEV/TEST/STAGING
- 浏览器：Chrome 120.0
- 操作系统：Windows 11

**复现步骤**：
1. 步骤1
2. 步骤2
3. 步骤3

**预期结果**：
- 预期行为

**实际结果**：
- 实际行为

**附件**：
- 截图
- 日志

**影响范围**：
- 影响的功能点

**建议解决方案**：
- 可能的解决方案
```

### 7.3 Bug跟踪流程

```
新建 → 分配 → 进行中 → 已修复 → 待验证 → 已关闭
         ↓                      ↓
      无法重现               重新打开
```
