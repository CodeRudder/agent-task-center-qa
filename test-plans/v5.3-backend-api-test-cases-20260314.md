## 🔗 P2-3: 任务依赖关系API测试用例

### TC-P2-3-API-001: 获取任务依赖关系列表

**API端点**: `GET /api/tasks/{taskId}/dependencies`

**请求参数**:
- taskId: 任务ID（路径参数）

**请求头**:
```
Authorization: Bearer {accessToken}
```

**预期响应**:
- 状态码: 200
- 响应体:
```json
{
  "success": true,
  "statusCode": 200,
  "message": "Success",
  "data": [
    {
      "id": "依赖关系ID",
      "taskId": "任务ID",
      "dependsOnTaskId": "依赖任务ID",
      "dependencyType": "blocking|sequential",
      "isBlocking": true|false,
      "createdAt": "创建时间",
      "updatedAt": "更新时间"
    }
  ]
}
```

**测试步骤**:
1. 使用有效的accessToken调用API
2. 验证响应状态码为200
3. 验证响应体包含success=true
4. 验证data数组包含依赖关系列表

---

## 📊 P2-5: 报表统计API测试用例

### TC-P2-5-API-001: 任务统计

**API端点**: `GET /api/statistics/tasks`

**请求参数**:
- startDate: 开始日期（可选）
- endDate: 结束日期（可选）

**预期响应**:
- 状态码: 200
- 响应体:
```json
{
  "success": true,
  "statusCode": 200,
  "message": "Success",
  "data": {
    "total": 总任务数,
    "byStatus": {
      "pending": 待办数量,
      "in_progress": 进行中数量,
      "completed": 已完成数量
    },
    "byPriority": {
      "high": 高优先级数量,
      "medium": 中优先级数量,
      "low": 低优先级数量
    },
    "completionRate": 完成率
  }
}
```
