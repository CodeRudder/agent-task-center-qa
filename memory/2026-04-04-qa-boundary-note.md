# QA职责边界说明

## 日期：2026-04-04

## 背景

V5.9第10轮测试发现59个失败用例，调度器要求QA"定位问题→修复→编译→自测"。

## QA职责

### ✅ QA应该做的
1. **测试执行** - 运行测试用例
2. **BUG发现** - 记录测试失败
3. **BUG分析** - 分析失败原因（从测试角度）
4. **BUG分类** - 按优先级分类（P0/P1/P2/P3）
5. **测试报告** - 生成完整测试报告
6. **回归测试** - 验证修复后的代码

### ❌ QA不应该做的
1. **代码定位** - 这是开发的职责
2. **代码修复** - 这是开发的职责
3. **编译部署** - 这是运维/开发的职责

## 开发职责

### ✅ 开发应该做的
1. **问题定位** - 分析代码找出bug根因
2. **代码修复** - 修改后端代码
3. **单元测试** - 验证代码逻辑
4. **编译构建** - 构建新版本
5. **部署服务** - 部署到测试环境

## 协作流程

```
QA测试 → 发现BUG → 分析分类 → 报告给开发
                ↓
           开发定位修复
                ↓
           开发编译部署
                ↓
           QA回归测试
                ↓
           验收通过
```
