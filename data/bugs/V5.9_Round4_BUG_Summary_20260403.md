### BUG-R4-001: 获取任务列表API返回500错误

**TC编号**：TC-TASK-R4-001
**API路径**：GET /api/v1/tasks
**优先级**：🔴 P0（阻塞核心功能）

**操作步骤**：
1. 使用测试账号登录系统：`admin@example.com / Admin123!`
2. 发送POST请求到 `/api/v1/auth/login` 获取访问令牌
3. 从登录响应中提取 `accessToken`
4. 发送GET请求到 `/api/v1/tasks`
5. 添加Authorization header：`Bearer ${accessToken}`

**输入数据/参数**：
```bash
# Headers
Authorization: Bearer ${accessToken}
Content-Type: application/json

# Query Parameters
page: 1
limit: 10
status: todo
```

**实际结果**：
- **状态码**：500
- **响应体**：
```json
{
  "success": false,
  "message": "Internal server error"
}
```

**期望结果**：
- **状态码**：200
- **响应体**：
```json
{
  "success": true,
  "data": {
    "tasks": [
      {
        "id": "uuid",
        "title": "任务标题",
        "status": "todo",
        "priority": "medium"
      }
    ],
    "pagination": {
      "page": 1,
      "limit": 10,
      "total": 100
    }
  }
}
```

**影响范围**：任务列表功能完全不可用，所有用户无法查看任务列表

**修复建议**：
1. 检查后端服务日志
2. 定位500错误的具体原因
3. 检查数据库查询逻辑
4. 检查业务逻辑异常
