## 🔴 P0级问题（必须修复）

### 1. 登录接口参数验证缺失
**影响**: 6个测试用例失败
**文件**: `tests/integration/api/auth-module/login.test.js`

**问题描述**:
- 缺少邮箱字段验证（预期400，实际401）
- 缺少密码字段验证（预期400，实际401）
- 邮箱格式未验证（预期400，实际401）
- 空字段未验证（预期400，实际401）

**失败用例**:
- ✗ 边界条件 - 缺少邮箱字段
- ✗ 边界条件 - 缺少密码字段
- ✗ 边界条件 - 邮箱格式错误
- ✗ 边界条件 - 空邮箱字段
- ✗ 边界条件 - 空密码字段
- ✗ 安全验证 - 密码错误不返回具体错误信息

**预期行为**:
```json
// 缺少必填字段应返回400
{
  "success": false,
  "statusCode": 400,
  "message": "邮箱是必填字段"
}
```

**实际行为**:
```json
// 实际返回401（认证失败）
{
  "success": false,
  "statusCode": 401,
  "message": "Unauthorized"
}
```

**修复建议**:
在登录接口增加请求体验证中间件：
```javascript
// 示例：backend/src/routes/auth.js
router.post('/login',
  validateLoginRequest, // 新增验证中间件
  authController.login
);

// 验证中间件
const validateLoginRequest = (req, res, next) => {
  const { email, password } = req.body;

  if (!email) {
    return res.status(400).json({
      success: false,
      message: '邮箱是必填字段'
    });
  }

  if (!password) {
    return res.status(400).json({
      success: false,
      message: '密码是必填字段'
    });
  }

  if (!isValidEmail(email)) {
    return res.status(400).json({
      success: false,
      message: '邮箱格式不正确'
    });
  }

  next();
};
```

### 2. 权限管理API响应格式问题
**影响**: 2个测试用例失败
**文件**: `tests/integration/api/permission-module/get-role-detail.test.js`

**问题描述**:
- 角色详情接口的`permissions`字段预期为数组，实际返回非数组类型

**失败用例**:
- ✗ 正常场景 - 获取角色详情成功
  - 预期: `response.body.data.permissions` 为数组
  - 实际: 返回 `false`（非数组）

**修复建议**:
确保角色详情接口返回的permissions字段为数组类型：
```javascript
// backend/src/controllers/roleController.js
const getRoleDetail = async (req, res) => {
  const role = await Role.findByPk(req.params.id, {
    include: [{ model: Permission, as: 'permissions' }]
  });

  // 确保permissions为数组
  const roleData = {
    ...role.toJSON(),
    permissions: role.permissions || [] // 确保返回空数组而非null/undefined
  };

  res.json({
    success: true,
    data: roleData
  });
};
```
