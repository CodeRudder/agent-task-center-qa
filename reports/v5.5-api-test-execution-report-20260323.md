### 4. 防止重复投票（TC-005）

**测试用例**：TC-005：防止重复投票

**测试步骤**：
1. 已投票状态下再次投票（相同类型）
2. 已投票状态下修改投票（不同类型）
3. 验证投票记录是否更新

**测试结果**：

**测试1：再次投票（相同类型：downvote）**
```json
{
  "success": true,
  "statusCode": 201,
  "data": {
    "id": "de14c116-cfe4-4de2-9e7e-e465be966d87",  // 同一个ID
    "voteType": "downvote",
    "createdAt": "2026-03-23T10:37:17.421Z",  // 创建时间不变
    "updatedAt": "2026-03-23T10:37:40.479Z"    // 更新时间改变
  }
}
```

**测试2：修改投票（改为upvote）**
```json
{
  "success": true,
  "statusCode": 201,
  "data": {
    "id": "de14c116-cfe4-4de2-9e7e-e465be966d87",  // 同一个ID
    "voteType": "upvote",  // 类型改变
    "createdAt": "2026-03-23T10:37:17.421Z",  // 创建时间不变
    "updatedAt": "2026-03-23T10:37:40.513Z"    // 更新时间改变
  }
}
```

**验证结论**：
- ✅ API设计是**允许用户修改投票**，而不是拒绝重复投票
- ✅ 每次投票都更新现有记录，而不是创建新记录
- ✅ 投票记录ID不变，只更新voteType和updatedAt
- ✅ 这是合理的设计决策，允许用户改变主意

**结论**：✅ 通过（设计符合预期）
